When a Third-Party Gateway Makes Sense

A third-party gateway is useful when you:

  • Do not have a Chinese phone number or Alipay account
  • Want to issue separate keys to team members, each with its own quota
  • Need hard quota caps — no surprise overages on your bill
  • Want per-key usage reports to see exactly who used how many tokens
  • Prefer clean output without reasoning/thinking tokens mixed in
  • Are building a product on top of DeepSeek and need predictable cost control

What This Gateway Is NOT

  • Not an official DeepSeek service or reseller
  • Not DeepSeek itself — this is an independent third-party service
  • Not open for public registration yet
  • Not a replacement for OpenRouter if you need 20+ different models

Token Counting

This gateway uses the usage.total_tokens value returned by DeepSeek for each request. Input tokens and output tokens are both counted at total_tokens in the current billing. This is a simplified model — the commercial version will use a fairer credit-based system that accounts for input/output/cache pricing differences.

Rate Limiting

Each key has its own configurable rate limit (default: 30 requests/minute). This means one user hammering the API does not slow down others. OpenRouter and direct DeepSeek API share rate limits across all requests from your account.
